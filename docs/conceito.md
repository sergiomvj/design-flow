
Objetivo do sistema:
Organizar pedidos de design, evitar retrabalho, centralizar briefing, arquivos, prazos, responsáveis, aprovações e status do projeto.

Nome da interface:
Design Request Control System

Conserve todos os elementos visuais do projeto original.

Estrutura da tela:
1. Sidebar lateral fixa com menu
2. Topbar superior com título do sistema, campo de busca e perfil do usuário
3. Área principal com dashboard e formulário de solicitação
4. Cards informativos com status dos projetos
5. Visualização clara e hierárquica das seções do formulário

Telas do sistema:
- Dashboard
- New Request
- Active Projects
- Waiting Approval
- Production Queue
- Completed Jobs
- Team
- Settings

Na tela principal, criar:
- Um cabeçalho com título “New Design Request”
- Subtítulo explicando que o formulário serve para centralizar e organizar todas as solicitações criativas
- Cards no topo mostrando:
  - Received Requests
  - In Creation
  - Waiting Approval
  - Approved
  - In Production
  - Completed

Abaixo, criar um Multi-step form dividido em blocos visuais bem organizados:

Bloco 1: Requester Information
Campos:
- Requester Name
- Company / Client
- Email
- Phone / WhatsApp
- Internal Sales Rep
- Department
- Preferred Contact Method

Bloco 2: Project Type
Campos:
- Type of Project
- Project Category
- Objective of the Piece
- New Design / Redesign / Adaptation / Revision

Bloco 3: Project Description
Campos:
- Detailed Project Description
- Where Will It Be Used
- Main Message
- Required Headline
- Required Text
- Mandatory Information

Bloco 4: Technical Specifications
Campos:
- Final Size
- Unit of Measurement
- Material
- Quantity
- Desired Colors
- Existing Brand Guidelines
- Special Finishing Details

Bloco 5: References and Files
Campos:
- Existing Logo
- Upload Area for Files
- Upload Area for Photos
- Reference Links
- What the Client Likes
- What the Client Does Not Want

Bloco 6: Deadline and Priority
Campos:
- Ideal Delivery Date
- Rush Job? ( bolean)
- Reason for rush
- Event / Installation Date

Bloco 7: Approval
Campos:
- Approver Name
- Approver Contact
- Number of Reviewers
- Number of Included Revisions
- Acknowledgment of Additional Charges After Approval

Bloco 8: Internal Control
Campos:
- Job Number
- Assigned Designer
- Internal Priority
- Project Status
- Internal Notes

Bloco 9: Final Confirmation
Campos:
- Information confirmed
- Aware of delays caused by incomplete briefing
- Authorization to start after internal validation

Requisitos de design:
- Cada bloco deve estar em cards com cantos suaves
- Excelente espaçamento
- Hierarquia tipográfica forte
- Labels claros
- Inputs modernos
- Selects elegantes
- Upload boxes com estilo drag-and-drop
- Botões modernos: Save Draft, Submit Request, Reset Form
- Indicadores visuais de progresso do preenchimento
- Microinterações sutis
- Layout com sensação de sistema real pronto para produção
- Mostrar exemplo de dados preenchidos em alguns campos para dar realismo
- Inserir pequenos ícones em cada seção
- Criar tabela ou painel lateral mostrando últimos projetos enviados

Extras desejados:
- Um painel lateral ou inferior com os últimos pedidos
- Tags coloridas para status
- Timeline pequena do fluxo: Received > Creating > Review > Approval > Production > Completed
- Visual que passe controle, organização e inteligência operacional


Formato desejado:
- Uma tela completa de dashboard + formulário
- UI/UX refinado
- Alta fidelidade
- Aspecto de produto digital real
